# PM2.5 Dashboard Technical Documentation

เอกสารนี้สรุปเงื่อนไขและลอจิกการทำงานของตัวกรอง (Filters) กราฟ (Charts) และโครงสร้างการจัดวางหน้าจอ (Layout Layer) ที่ใช้งานอยู่ในหน้า `src/app/dashboard/pm25/page.tsx`

## 1. เงื่อนไขและพฤติกรรมของตัวกรอง (Filters)

ตัวกรองข้อมูลถูกออกแบบให้ทำงานแบบ Cascading (สัมพันธ์และส่งผลกระทบต่อกันเป็นลำดับขั้น) โดยมีเงื่อนไขดังนี้:

### 1.1 ตัวกรองวันที่ (จากวันที่ - ถึงวันที่)
- **ข้อมูลตัวเลือก**: ดึงรายการวันที่ทั้งหมดจาก API และจะถูกกรองเอาเฉพาะวันที่ **ไม่เกินวันปัจจุบัน (Today)** เท่านั้น
- **ค่าเริ่มต้น (Default Value)**: 
  - `ถึงวันที่` (endDate): จะเลือกวันที่ล่าสุดที่มีข้อมูล
  - `จากวันที่` (startDate): จะถูกคำนวณจาก `ถึงวันที่` โดยย้อนหลังกลับไป 6 วัน (ให้ครอบคลุม 7 วัน) หากไม่มีข้อมูลในวันนั้น จะใช้วันที่เก่าที่สุดที่มีแทน

### 1.2 ตัวกรองระดับพื้นที่ (เขตสุขภาพ -> จังหวัด -> อำเภอ/เขต)
- **เขตสุขภาพ (Regions)**:
  - การเลือกหรือเปลี่ยนเขตสุขภาพ จะส่งผลให้ **ล้างค่า (Clear)** ข้อมูลที่เลือกไว้ในตัวกรอง "จังหวัด" ทิ้งไปโดยอัตโนมัติ
- **จังหวัด (Provinces)**:
  - **ตัวเลือกที่มีให้เลือก (Options)**: จะแสดงรายการจังหวัดเฉพาะที่สังกัดอยู่ใน "เขตสุขภาพ" ที่เลือกไว้เท่านั้น (หากไม่ได้เลือกเขตสุขภาพใดๆ เลย จะแสดงรายการจังหวัดทั้งหมดในประเทศไทย)
  - การเลือกหรือเปลี่ยนจังหวัด จะส่งผลให้ **ล้างค่า (Clear)** ข้อมูลที่เลือกไว้ในตัวกรอง "อำเภอ/เขต" ทิ้งไปโดยอัตโนมัติ
- **อำเภอ/เขต (Districts)**:
  - **ตัวเลือกที่มีให้เลือก (Options)**: จะแสดงรายการอำเภอเฉพาะที่สังกัดอยู่ใน "จังหวัด" ที่เลือกไว้เท่านั้น (หากยังไม่ได้เลือกจังหวัดใดๆ เลย ตัวเลือกในช่องอำเภอจะว่างเปล่า)

## 2. เงื่อนไขและพฤติกรรมของการแสดงผลกราฟ (Charts)

คอมโพเนนต์ `MultiLineChart` สำหรับแสดงกราฟแนวโน้มมีลอจิกและลูกเล่นในการแสดงผลดังนี้:

### 2.1 การเรียงลำดับรายชื่อข้อมูล (Label Sorting)
รายชื่อที่ปรากฏใน Legend (ด้านขวาของกราฟ) จะถูกจัดเรียงตามเงื่อนไขของชื่อกราฟโดยอัตโนมัติ:
- **กราฟระดับเขตสุขภาพ** (ชื่อกราฟมีคำว่า "เขต"): 
  - จัดเรียงตามตัวเลขที่อยู่ในชื่อจากน้อยไปมาก (เช่น เขต 1 -> เขต 2 -> ... -> เขต 12)
  - หากมีชื่อพื้นที่ **"กรุงเทพ"** จะถูกผลักไปไว้เป็นลำดับสุดท้ายเสมอ
- **กราฟระดับจังหวัดและอำเภอ**: 
  - จัดเรียงตามตัวอักษรภาษาไทย **ก-ฮ** (Thai Alphabetical Order)

### 2.2 การโต้ตอบกับผู้ใช้ (Interactive Legend)
- ผู้ใช้สามารถ **คลิกที่รายชื่อ** บริเวณแถบ Legend เพื่อ **เปิด/ปิด** การแสดงผลเส้นกราฟของพื้นที่นั้นๆ ได้
- เมื่อถูกปิด เส้นกราฟในจอจะหายไป และรายชื่อใน Legend รวมถึงจุดสีจะเปลี่ยนเป็นสีเทาจาง (Opacity ลดลง)

### 2.3 การนับจำนวนรายการข้อมูล (Label Count)
- บริเวณหลังชื่อของกราฟแต่ละตัว จะมีตัวเลขแสดงอยู่ในกรอบสีฟ้า (เช่น `(77)`) เพื่อบอกให้ผู้ใช้ทราบว่ากราฟนี้กำลังแสดงเส้นข้อมูลทั้งหมดกี่พื้นที่ ณ ขณะนั้น (ซึ่งสัมพันธ์กับการเลือกตัวกรอง)

### 2.4 แกนแนวตั้งและการพล็อตข้อมูล (Y-Axis & Data Plotting)
- **Scale (Y-Axis Maximum)**: จะคำนวณอัตโนมัติจากค่าสูงสุด (Max Value) ในบรรดาเส้นทั้งหมดที่ถูกวาดอยู่บนกราฟ โดยบวกเพิ่มพื้นที่ว่างด้านบนอีก `10%` เพื่อไม่ให้เส้นกราฟชนขอบบนสุด (หากไม่มีข้อมูลจะตั้งค่าเริ่มต้นแกนไว้ที่ 100)
- จุดข้อมูลของแต่ละเส้น จะถูกนำมาจัดเรียงตามลำดับเวลา (Chronological order) ก่อนทำการวาดเส้น เพื่อให้เส้นกราฟต่อเนื่องกันอย่างถูกต้อง

## 3. แผนที่รายงานระดับค่าฝุ่น (Thailand Map)

แผนที่มีการแสดงผลแบบมี Tab ให้ผู้ใช้กดเลือกสลับดูข้อมูลได้ 3 มุมมองหลัก ซึ่งมีลอจิกและเงื่อนไขการคำนวณที่แตกต่างกัน:

### 3.1 โหมด "ค่าฝุ่น PM2.5" (Maximum PM2.5)
- **เงื่อนไขที่ใช้**: นำค่าฝุ่น PM2.5 ของทุกๆ วันใน **"ช่วงวันที่คุณเลือกทั้งหมด"** มาหาค่าที่สูงที่สุด (Max Value)
- **การแสดงผล**: แผนที่จะพล็อตข้อมูลด้วยค่าฝุ่นสูงสุดที่พบในจังหวัดนั้นๆ ตามเกณฑ์มาตรฐานสี (เขียว, เหลือง, ส้ม, แดง) โดยมีหน่วยใน Popup เป็น `มคก./ลบ.ม.`
- **สิ่งที่บอก**: ภาพรวมความรุนแรงของฝุ่นที่พุ่งไปถึงจุดสูงสุดในจังหวัดนั้นๆ ตลอดช่วงเวลาที่ถูกกรอง

### 3.2 โหมด "เกิน 37.5" (Active Streak > 37.5)
- **เงื่อนไขที่ใช้**: ดูสถานะฝุ่น **"ณ วันสุดท้ายที่คุณเลือก (ถึงวันที่)"** เป็นหลัก
  - ถ้าน้อยกว่า 37.5: จะถือว่าปลอดภัย (เป็น 0 วัน) และไม่แจ้งเตือน
  - **ถ้าเกิน 37.5**: ระบบจะเริ่ม **"นับถอยหลังทีละวัน"** ตามปฏิทินจากวันสุดท้ายกลับไปในอดีต เพื่อดูว่ามันเกินติดต่อกันมากี่วันแล้ว (คำนวณหาสถานะปัจจุบันที่กำลังดำเนินอยู่)
- **การแสดงผล**: โชว์จำนวนวันที่เกินติดต่อกัน (เช่น 7 วัน) และโชว์ช่วงวันที่ล่าสุดที่นับได้จากจุดสิ้นสุด (เช่น `26 มี.ค. - 1 เม.ย.`)
- **สิ่งที่บอก**: ปัจจุบัน (ณ วันที่เลือกในตัวกรอง) จังหวัดนี้กำลังมีวิกฤตฝุ่นเกินมาตรฐานติดต่อกันอยู่หรือไม่ และลากยาวมากี่วันแล้ว

### 3.3 โหมด "เกิน 75" (Latest Historic Streak > 75)
- **เงื่อนไขที่ใช้**: ค้นหาช่วงเวลาที่ค่าฝุ่นพุ่งสูงกว่า 75 **"ติดต่อกันตั้งแต่ 2 วันขึ้นไป"** 
- **จุดต่างจาก 37.5**: ระบบจะไม่สนใจว่าวันสุดท้าย (ถึงวันที่) ฝุ่นจะเกินหรือไม่ แต่มันจะไปกวาดหาประวัติ **ช่วงที่เกิน 75 ติดต่อกัน "ครั้งล่าสุด"** ที่เกิดขึ้นที่ใดก็ได้ในช่วงเวลาที่คุณกรองไว้แทน
- **การแสดงผล**: โชว์จำนวนวันที่เคยเกินติดต่อกัน (เช่น 3 วัน) และโชว์ช่วงวันที่เกิดเหตุการณ์วิกฤตนั้น (เช่น `10 ก.พ. - 12 ก.พ.`)
- **สิ่งที่บอก**: ในช่วงเวลาที่ดูข้อมูล จังหวัดนี้เคยมีวิกฤตฝุ่นสีแดงแบบรุนแรงต่อเนื่องเกิดขึ้นบ้างหรือไม่ และเหตุการณ์นั้นเกิดขึ้นในช่วงวันที่เท่าไหร่

## 4. โครงสร้างการจัดวางหน้าจอ (Layout & UI Positioning)

หน้าเว็บถูกออกแบบแบบ **Single Page Application (SPA)** ที่ไม่มีการเลื่อนหน้าลง (No Vertical Scroll on Body) โดยใช้ความสูง 100vh ตลอดเวลา โครงสร้างหลัก (Layout Layer) แบ่งออกเป็น 4 ส่วน ดังนี้:

### 4.1 Global Layout (`<div className="min-h-screen...">`)
- **พื้นหลัง (Background)**: ใช้รูปภาพ `/img/background.jpg` เป็นภาพพื้นหลังแบบปกคลุมเต็มจอ (cover) และฟิกซ์ตำแหน่ง (fixed)
- **เลเยอร์ซ้อนทับ (Overlay)**: มี `<div className="absolute inset-0 bg-slate-900/40 z-0">` เพื่อดรอปความสว่างของรูปภาพลง 40% ทำให้ข้อความด้านหน้าโดดเด่นขึ้น
- **คอนเทนเนอร์หลัก (`<main>`)**: กำหนดความกว้างสูงสุดที่ `max-w-screen-2xl` ความสูง `h-screen` และบังคับการจัดเรียงเป็นแนวตั้ง `flex flex-col gap-4`

### 4.2 ส่วนหัว (Header Layer - `z-[60]`)
- **ความสูง**: ไม่ถูกบีบ (`shrink-0`)
- **องค์ประกอบ**: 
  - **ซ้าย**: โลโก้ DDC (กรมควบคุมโรค) วางในกล่องสีขาวมีขอบและเงา (ring-4), ตามด้วยชื่อระบบและชื่อหน่วยงาน (Drop shadow)
  - **ขวา**: ปุ่มสี่เหลี่ยมไอคอน Home สำหรับย้อนกลับหน้าหลัก

### 4.3 ส่วนตัวกรองข้อมูล (Filter Panel - `z-[100]`)
- **ดีไซน์**: กล่องยาวขวาง (Glassmorphism: `bg-white/10 backdrop-blur-2xl`) ขอบมน `rounded-3xl`
- **โครงสร้าง Grid**: แบ่งเป็นคอมลัมน์อัตโนมัติตามขนาดจอ 
  - มือถือ (`sm`): 1 แถว / 1 คอลัมน์
  - แท็บเล็ต (`md`): 3 คอลัมน์
  - จอใหญ่ (`xl`): 6 คอลัมน์ (แสดงตัวกรองเรียงต่อกันเป็นแนวยาว)
- **ลำดับตัวกรอง**: 1. จากวันที่ -> 2. ถึงวันที่ -> 3. เขตสุขภาพ -> 4. จังหวัด -> 5. อำเภอ

### 4.4 ส่วนข้อมูลสรุป (Summary Cards - `z-[90]`)
- **ดีไซน์**: แสดงการ์ด 4 ใบเรียงต่อกัน (Grid)
- **โครงสร้าง Grid**: 1 คอลัมน์ (มือถือ) -> 2 คอลัมน์ (แท็บเล็ต) -> 4 คอลัมน์ (จอใหญ่)
- **การ์ดใบแรก (Primary)**: พื้นหลังไล่เฉดสี `bg-linear-to-br from-blue-600/90 to-sky-500/90` เน้นความสำคัญ (ค่าเฉลี่ย 24 ชม.)
- **การ์ดใบอื่นๆ (Secondary)**: พื้นหลังใส `bg-white/10` เน้นตัวเลข และมีแทบสี (indicator) ด้านหลังตัวเลขตามระดับความรุนแรง
- **ระบบ Tooltip**: การ์ดลำดับที่ 3 และ 4 (เกินมาตรฐาน) จะมี Tooltip ที่ซ่อนอยู่ (`opacity-0 invisible`) และจะปรากฏขึ้นเมื่อ Hover เนื้อหาใน Tooltip เป็น Grid แสดงรายชื่อจังหวัดแยกตามเขตสุขภาพ

### 4.5 ส่วนกราฟและแผนที่ (Content Layer - `z-[10]`)
- **ความสูง**: กินพื้นที่ที่เหลือทั้งหมด (`flex-1 min-h-0`)
- **โครงสร้าง Grid**: 
  - จอเล็ก-กลาง: แสดงเรียงแนวตั้งแบบ 1 คอลัมน์
  - จอใหญ่ (`lg:grid-cols-[1.6fr_1.1fr]`): แบ่ง 2 คอลัมน์ สัดส่วน **กราฟ (1.6) : แผนที่ (1.1)**
- **บล็อกซ้าย (กราฟแนวโน้ม)**: 
  - มี scrollbar ภายในตัวเอง (`overflow-y-auto custom-scrollbar`) เพื่อให้สามารถดู MultiLineChart 3 ตัว (เขต, จังหวัด, อำเภอ) ได้โดยไม่ต้องเลื่อนหน้าเว็บหลัก
- **บล็อกขวา (แผนที่)**: 
  - ด้านบน: หัวข้อแผนที่ + แท็บสลับโหมด 3 โหมด (ค่าฝุ่นPM2.5, เกิน 37.5, เกิน 75)
  - ด้านล่าง: 컴โพเนนต์ `ThailandMap` แสดงผลแผนที่เต็มพื้นที่กรอบ

### 4.6 CSS Override (`<style>`)
- ซ่อน Footer ของระบบ (`footer { display: none !important; }`)
- ปรับแต่ง Scrollbar ทั่วทั้งหน้าให้มีขนาดเล็กลง โปร่งใส และกลมกลืนกับพื้นหลังเข้ม
