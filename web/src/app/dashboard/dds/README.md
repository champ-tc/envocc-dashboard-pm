# DDS Dashboard Technical Documentation

เอกสารนี้สรุปเงื่อนไข ลอจิกการทำงาน และโครงสร้างการจัดวางหน้าจอ (Layout Layer) ของหน้า `src/app/dashboard/dds/page.tsx`

## 1. เงื่อนไขและพฤติกรรมของตัวกรอง (Filters)

ระบบตัวกรองถูกออกแบบให้มีความสัมพันธ์กันแบบ Cascading เพื่อความถูกต้องของข้อมูลเชิงพื้นที่:

### 1.1 ตัวกรองวันที่ (จากเดือน - ถึงเดือน)
- **ข้อมูลตัวเลือก**: แสดงรายการปี-เดือน ที่ดึงมาจากฐานข้อมูล DDS (Surveillance) โดยเรียงจากล่าสุดไปอดีต
- **ค่าเริ่มต้น (Default)**: 
  - `ถึงเดือน`: เดือนล่าสุดที่มีข้อมูลในระบบ
  - `จากเดือน`: คำนวณจากปีงบประมาณ (Fiscal Year) เริ่มต้นที่วันที่ `1 ตุลาคม` ของปีที่สอดคล้องกับเดือนล่าสุด

### 1.2 ตัวกรองพื้นที่ (Cascading Filters)
- **เขตสุขภาพ (Regions)** -> **จังหวัด (Provinces)** -> **อำเภอ/เขต (Districts)** -> **ตำบล/แขวง (Subdistricts)**
- หากเปลี่ยนค่าในระดับบน (เช่น เขต) ระบบจะทำการ **ล้างค่า (Clear)** ในระดับที่ต่ำกว่าทั้งหมดโดยอัตโนมัติ เพื่อป้องกันการเลือกพื้นที่ที่ไม่ได้สังกัดอยู่จริง

### 1.3 ตัวกรองประเภทการวินิจฉัย (Diagnosis Types)
มีการกรองข้อมูลตามกลุ่มรหัส ICD-10 3 รูปแบบหลัก:
1. **พ.ร.บ.EnvOcc ร่วมกับ Z58.1**: กรองรหัสเฉพาะที่กำหนดตามกฎหมายและสิ่งแวดล้อม
2. **พ.ร.บ.EnvOcc ร่วมกับ Z58.1+Y97**: เพิ่มรหัส Y97 เข้ามาในการวิเคราะห์
3. **Z58.1 ร่วมกับกลุ่มโรคที่ต้องการ**: โหมดที่อนุญาตให้ผู้ใช้ **เลือกรายรหัส ICD-10** ได้เองภายในการ์ดกลุ่มโรค

## 2. โครงสร้างการแสดงผลข้อมูล (Data Visualization)

### 2.1 ส่วนสรุปสถิติ (Stats Layer)
- **จำนวนผู้ป่วยทั้งหมด**: นับจาก `person_id` ที่ไม่ซ้ำกัน (Unique Patients)
- **จำนวนการวินิจฉัยทั้งหมด**: นับจำนวนรายการวินิจฉัยทั้งหมดที่ตรงตามเงื่อนไข
- **Disease Group Cards**: แบ่งเป็น 5 กลุ่มโรคหลัก (Respiratory, Circulatory, Skin, Eye, Health Status)
  - ในโหมดกำหนดเอง ผู้ใช้สามารถเลือก/ไม่เลือก รหัส ICD-10 เฉพาะรหัสที่ต้องการได้จากการ์ดเหล่านี้

### 2.2 กราฟแนวโน้ม (Monthly Trend Chart)
- **Stacked Bar Chart**: แสดงจำนวนผู้ป่วยแยกตามสีของกลุ่มโรค
- **PM2.5 Line Chart**: แสดงค่าเฉลี่ยฝุ่นในเดือนนั้นๆ ทับซ้อนบนกราฟแท่ง
- **Hover Detail**: แสดง Pop-up รายละเอียดจำนวนผู้ป่วยแยกรายกลุ่มโรค และค่าฝุ่นที่แม่นยำ

### 2.3 แผนที่รายงาน (Thailand Map)
- **สลับมุมมอง**: 
  - ระดับจังหวัด: แสดงภาพรวมความหนาแน่นของผู้ป่วยรายจังหวัด
  - ระดับตำบล: แสดงผลเป็นจุดพิกัดสถานี (Stations) หรือพื้นที่ตำบล (Detailed View) เมื่อมีการเลือกอำเภอ/เขต
- **สูตรคำนวณ**: จำนวนผู้ป่วยหารด้วยประชากร x 100,000 (อัตราป่วยต่อแสน)

## 3. โครงสร้างเลเยอร์และการจัดตำแหน่ง (Layout & Positioning)

### 3.1 Positioning Layers (z-index)
- **Header Layer (`z-[60]`)**: บาร์ด้านบนแสดงโลโก้ 2 หน่วยงาน และสถานะพื้นที่ของผู้ใช้งาน (Admin Role)
- **Filter Layer (`z-[50]`)**: แถบตัวกรอง Glassmorphism 7 คอลัมน์ (ในจอใหญ่)
- **Stats Layer (`z-[30]`)**: การ์ดตัวเลขหลักและการ์ดกลุ่มโรค
- **Visualization Layer (`z-[20]`)**: พื้นที่แสดงผลกราฟ (1.6fr) และแผนที่ (1.1fr)

### 3.2 UI Design (Glassmorphism)
- ใช้พื้นหลังโปร่งแสง `bg-white/10` ร่วมกับ `backdrop-blur-2xl`
- เส้นขอบสีขาวจาง `border-white/20` และเงา `shadow-2xl`
- ปุ่ม Login/Home และตัวเน้นใช้ Gradient `bg-linear-to-br from-blue-600 to-sky-500`

### 3.3 การจัดการพื้นที่ (Responsive)
- จอขนาดกลาง (`md`): สรุปสถิติจะแบ่งเป็น 2 คอลัมน์
- จอขนาดใหญ่ (`xl`): ตัวกรองจะเรียงเป็น 7 คอลัมน์แนวนอน เพื่อประหยัดพื้นที่แนวตั้ง
- จอมือถือ: ทุกอย่างจะเรียงตัวเป็น 1 คอลัมน์แนวตั้ง
